# K线分析工具

这是一个用于分析加密货币K线数据的Python脚本，主要用于计算特定时间点之前的交易量指标。

## 功能

1. 计算`vol_base`：截止时间往前最近的120根分钟K线中，排除最近15根后的平均交易量
   - 即T(15)到T(119)的平均交易量（当有足够数据时，总共105根）
   
2. 计算最近3根5分钟K线的交易量（如果有的话）：
   - q1 = T(0) + T(1) + T(2) + T(3) + T(4)
   - q2 = T(5) + T(6) + T(7) + T(8) + T(9)  
   - q3 = T(10) + T(11) + T(12) + T(13) + T(14)

其中T(0)是截止时间的那根K线，T(1)是截止时间前一根K线，以此类推。

## 使用方法

```bash
python3 kline_analyzer.py <kline_file> <cutoff_time>
```

### 参数说明

- `kline_file`：包含K线数据的文件路径
- `cutoff_time`：截止时间，ISO格式（例如：2026-04-28T23:00:00 或 2026-04-29T00:55:59.999Z）

### 示例

```bash
python3 kline_analyzer.py ./data/BROCCOLI714USDT_150.kline "2026-04-28T23:15:00"
```

## 输出说明

脚本会输出以下信息：

1. 总共加载的K线数量和类型统计
2. 用于计算vol_base的分钟K线数量和详细信息
3. vol_base值（T(15)到T(N)的平均交易量，N取决于可用数据）
4. 最近3根5分钟K线的交易量和总交易量（如果有）

## 文件格式要求

输入文件应该是包含JSON格式K线数据的文本文件，每根K线占一行或多行。支持的K线类型包括：
- `kline_1m`：1分钟K线
- `kline_5m`：5分钟K线

## 注意事项

- 脚本会自动处理带时区和不带时区的时间格式
- 如果没有足够的历史数据，计算结果可能会基于较少的K线数量
- 脚本会忽略无法解析的K线数据行
- 只有截止时间之前的数据会被用于计算
